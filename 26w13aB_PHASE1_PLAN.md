# OpenOxygen 26w13aB Phase1 开发计划

## 目标
修复 Release 包的部署问题，确保用户能够一键安装并正常运行。

## 问题清单

### P0: 关键问题 (阻塞用户使用)
- [ ] **端口冲突** - EADDRINUSE 错误，需要自动检测并切换端口
- [ ] **Native 模块缺失** - Release 包中 .node 文件未正确包含
- [ ] **配置文件路径错误** - 默认路径指向 C:\Users\Sails\.openoxygen\ 而非安装目录

### P1: 权限问题
- [ ] **Auth failure** - 127.0.0.1 认证失败计数
- [ ] **权限不足提示** - 用户看到权限错误但不知如何解决

### P2: 用户体验
- [ ] **启动脚本改进** - 更好的错误提示和自动修复
- [ ] **日志可见性** - 用户无法看到 .state\ 目录中的日志
- [ ] **安装验证** - 安装后自动运行健康检查

## 开发步骤

### Step 1: 修复 Release 打包脚本
- 确保 .node 文件被正确包含
- 确保 index.js 和 package.json 完整
- 添加 .gitattributes 保持 LF/CRLF 一致

### Step 2: 端口自动检测与切换
- 启动前检测 4800 是否被占用
- 如果被占用，自动尝试 4801, 4802...
- 更新 openoxygen.json 并提示用户

### Step 3: 配置文件路径修复
- 默认使用当前工作目录而非用户目录
- 添加 OPENOXYGEN_CONFIG_PATH 环境变量支持
- 启动脚本自动设置环境变量

### Step 4: 权限认证修复
- 检查 auth.mode 配置
- 默认 none 模式下不应有认证失败
- 修复 IP 白名单逻辑

### Step 5: 启动脚本增强
- 端口冲突检测和提示
- 自动尝试备用端口
- 显示清晰的错误信息
- 提供故障排除链接

### Step 6: 安装验证测试
- 创建安装后测试脚本
- 验证所有组件正常工作
- 生成安装报告

## 验收标准
- [ ] 全新 Windows 机器上下载 Release 包能正常启动
- [ ] 端口被占用时自动切换并提示
- [ ] Native 模块正确加载（无 OUV 警告）
- [ ] 无权限认证错误
- [ ] Dashboard 能正常访问
- [ ] LLM 推理能正常工作

## 时间估计
- Step 1-2: 30 分钟
- Step 3-4: 30 分钟
- Step 5-6: 30 分钟
- 测试验证: 30 分钟
- **总计: 2 小时**
